# 聊天历史列表、分组顶部设置按钮与知识库新建弹层优化

- 日期：2026-05-15
- 类型：前端交互修复
- 范围：
  - `frontend/src/components/AppSidebar.jsx`
  - `frontend/src/App.jsx`
  - `frontend/src/pages/knowledge/KnowledgePage.jsx`

## 用户反馈

1. 智能问答历史对话列表只能显示少量条目，其余历史对话既没有展示，也没有明确折叠入口
2. 单条对话在默认状态下会被右侧隐藏按钮预留空间挤压，导致标题更容易省略截断
3. 知识库、数据库、技能库、模型库的分组“设置”入口应移到页面顶部栏最右侧，仅保留图标
4. 知识库管理页的“新知识库名称 / 共享 / 新建”应合并为蓝色“新建”按钮，点击后使用弹出子页填写名称和共享选项

## 修复说明

## 1. 智能问答历史列表显示不全

在 `frontend/src/components/AppSidebar.jsx` 中：

- 移除了会话列表前端二次截断逻辑：
  - 不再先只渲染前 10 条
  - 已加载到前端的会话现在全部显示
- 保留后端分页加载机制：
  - 仍通过 `hasMore + 加载更多` 控制增量加载

这样可避免出现：

- 已经加载到前端的历史会话被截断不显示
- 但界面上又没有明确“展开全部”入口的异常体验

## 2. 单条对话标题被隐藏按钮挤压

在 `frontend/src/components/AppSidebar.jsx` 中：

- 将置顶 / 重命名 / 删除按钮从普通流布局改为右侧绝对定位浮层
- 默认状态下按钮完全不占标题空间
- hover 时才显示操作区，并通过标题 `pr` 过渡让出空间

结果：

- 默认状态标题能完整占满可用宽度
- hover 时操作按钮再出现
- 不再因为隐藏按钮预留宽度而提前截断标题

## 3. 分组设置按钮上移到页面顶部栏

### 数据库 / 技能库 / 模型库

在 `frontend/src/App.jsx` 中：

- 将分组设置入口从分组导航条中移除
- 放到各页面顶部栏最右侧
- 改为纯图标按钮：
  - 普通态显示 `Settings`
  - 设置态显示 `CheckSquare`

分组导航栏现在只负责：

- 展示已有分组
- 在设置态下显示“新建分组”

### 知识库

在 `frontend/src/pages/knowledge/KnowledgePage.jsx` 中：

- 同样将设置入口迁移到顶部栏最右侧
- 同样改为纯图标按钮
- 保持与数据库/技能库/模型库一致的切换逻辑：
  - 普通态：齿轮图标
  - 设置态：保存图标

## 4. 知识库新建入口改为蓝色按钮 + 弹层

在 `frontend/src/pages/knowledge/KnowledgePage.jsx` 中新增：

- `CreateDatasetModal`

并将原来顶部行内的：

- 新知识库名称输入框
- 共享勾选
- 黑色“新建”按钮

改为：

- 蓝色 `新建` 按钮
- 点击后弹出“新建知识库”弹层
- 在弹层中填写：
  - 知识库名称
  - 是否共享

颜色风格对齐数据库页的“添加连接”按钮。
